논리적 데이터베이스 설계
- 개념적 설계의 산물인 ER스키마에서 관계스키마로 사상(mapping rule)
- 효율적인 데이터베이스 스키마 구축
 -> 정규화(Namalization)
    키, 무결성의 개념
    이상현상 이해
    함수적 종속성 
    정규화 과정

이상(Anomaly)
: 데이터 중복으로 인해 릴레이션 조작시 예상하지 못한 곤란한 현상이 발생함
- 삽입 이상
- 삭제 이상
- 갱신 이상

함수적 종속성
- 완전 함수적 종속(2정규화)
: 기본키가 복합키로 구성되어 있을때, 기본키가 결정자가 되어서 속성값이 나온다
- 부분 함수적 종속(2정규화)
: 기본키가 복합키로 구성되어 있을때, 기본키의 일부가 결정자가 되어서 속성값이 나온다.
- 이행 함수적 종속(3정규화)
: A -> B -> C
- 다치 종속(4정규화)
: 집합으로 중복될때
- 조인 종속(5정규화)
: 조인으로 위조튜플이 나오면 안된다 나눠진 테이블을 조인했을 때 원래의 테이블과 같아야한다. 하나의 테이블을 더 추가해서 조인함
